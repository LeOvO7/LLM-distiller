# LLM-distiller
A demonstration of **offline SFT (Supervised Fine-Tuning) distillation** where a large model transfers knowledge of a made-up programming language to a tiny model.

## The Floq Language

Floq is a fictional language designed to have no overlap with any real language the student model has seen during pre-training. This makes it an ideal controlled domain for measuring knowledge transfer.

```floq
## Compute factorial
fn factorial(n) =>
  when n <= 1 { 1 } else { n * factorial(n - 1) }

val result := factorial(5)
shout(result)  ## prints 120
```

### Syntax Rules

| Feature | Syntax |
|---|---|
| Variable declaration | `val x := 42` |
| Function definition | `fn add(a, b) => a + b` |
| Conditional | `when x > 0 { ... } else { ... }` |
| Loop | `loop 10 times { ... }` |
| Print / Output | `shout("hello")` |
| Types | `num`, `str`, `bool`, `list` |
| Comments | `## This is a comment` |
| List literal | `val xs := [1, 2, 3]` |
| List indexing | `xs~0` (tilde `~` replaces `[]`) |
| String concatenation | `"hello" ++ " world"` |
| Boolean operators | `and`, `or`, `not` |

## Pipeline

```
Step 0  Install dependencies
Step 1  Load API keys (OpenAI + Anthropic) & verify GPU
Step 2  Define Floq spec + generate training dataset (teacher: GPT-4o-mini)
        + generate structured tiered test set (40 items across 4 difficulty tiers)
Step 3  Evaluate baseline student (Qwen2.5-0.5B, no training)
        + Claude LLM-as-Judge evaluation (tiered)
Step 4  LoRA fine-tuning on 200 teacher-generated samples
Step 5  Evaluate distilled student
        + Claude LLM-as-Judge evaluation (tiered)
Step 6  Visualize results (keyword by tier, judge scores, difficulty curve, heatmap)
Step 7  Generate interactive HTML report
```

## Dataset

### Training Set
- **200 items** generated by GPT-4o-mini from randomized question templates
- Categories: syntax queries, code comprehension, invalid-code detection, full program writing
- Saved as `floq_data/train.json`

### Test Set (Tiered)
The test set consists of **40 fixed questions** across four difficulty tiers, replacing the original random 30-item set. Fixed questions ensure reproducibility and enable meaningful cross-run comparison.

| Tier | Size | What it tests |
|---|---|---|
| **Easy** | 10 | Single-feature recall — keywords, types, one-liner syntax |
| **Medium** | 10 | Multi-feature composition — functions + conditionals + loops |
| **Hard** | 10 | Complex programs — recursion, nested logic, multi-function composition |
| **OOD** | 10 | Out-of-distribution — Python/JS→Floq translation, error debugging, open-ended design |

Saved as `floq_data/test_tiered.json`.

## Evaluation

### Keyword Hit Rate
Measures what fraction of 11 Floq-specific syntax tokens appear in the model's response:

```
:=   fn    =>   when   shout(   loop   times   ~   val   ++   ##
```

A score of 1.0 means all 11 keywords appeared in the answer. This directly measures whether the model uses Floq syntax rather than falling back to Python or JavaScript conventions. However, it systematically underestimates quality — a fully correct, concise answer may only use 3–4 keywords.

### LLM-as-Judge (Claude)
To address keyword hit rate's blind spots, each student response is additionally scored by `claude-haiku-4-5` across four dimensions:

| Dimension | Max | What it measures |
|---|---|---|
| `syntax_correctness` | 3 | Correct use of Floq syntax constructs |
| `no_leakage` | 2 | Absence of Python/JS syntax contamination |
| `completeness` | 2 | Whether the question is fully answered |
| `executability` | 3 | Whether the code would actually run in Floq |

**Total: 10 points per response.** The judge score is the more reliable indicator of actual capability — the distilled model's jump from 3.8% → 77.5% reflects genuine syntactic competence that keyword counting alone misses.

## LoRA Configuration

```python
LoraConfig(
    r=32,                  # rank
    lora_alpha=64,         # scaling factor
    target_modules=[       # layers injected with LoRA
        "q_proj", "k_proj", "v_proj", "o_proj",
        "gate_proj", "up_proj", "down_proj"
    ],
    lora_dropout=0.05,
    task_type=TaskType.CAUSAL_LM
)
```

- **Trainable parameters**: 17,596,416 out of 511,629,184 (3.44%)
- **Training time**: ~57.6 seconds on A100
- **Final training loss**: 0.4497

### Training Hyperparameters

| Parameter | Value |
|---|---|
| Epochs | 5 |
| Effective batch size | 16 (4 × 4 accumulation steps) |
| Learning rate | 2e-4 |
| LR scheduler | Cosine |
| Optimizer | `paged_adamw_32bit` |
| Precision | FP16 |
| Max sequence length | 512 |
